Multicharts System Optimization Procedure

Metadata

Category: Multicharts Models Last Updated: 2024-01-15 Author: System Administrator Version: 1.0 Review Date: 2024-04-15 Status: Active

Purpose

This procedure covers optimizing Multicharts system for running 40+ instances, including GDI handle limits, memory allocation, and process management.

When to Use: - Initial system optimization - Performance degradation issues - Scaling to more instances - System tuning

Prerequisites

Required Access

  • [x] Administrator privileges
  • [x] Access to Multicharts system

Required Tools

  • [x] PowerShell (Administrator)
  • [x] Script: Scripts\02-Multicharts-Models\Optimized-Multicharts-Management.ps1
  • [x] Backup script (run first!)

Procedure Steps

Step 1: Backup Current State

Action:

cd I:\HyperV_PowerScript_Optimazation\Scripts\02-Multicharts-Models
.\Backup-Multicharts-State.ps1

Expected Result: - Backup created successfully

Step 2: Apply Settings Only

Action:

.\Optimized-Multicharts-Management.ps1 -ApplySettingsOnly

Expected Result: - Registry settings applied - System settings configured

Step 3: Reboot System

Action:

# MANDATORY: Reboot required
Restart-Computer -Force

Expected Result: - System reboots - Settings take effect

Step 4: Enable Monitoring

Action:

# After reboot, enable monitoring
.\Optimized-Multicharts-Management.ps1 -EnableMonitoring -MaxMonitorIterations 288

Expected Result: - Monitoring enabled - Performance metrics tracked
