Best Practices for SOP Documentation

1. Use Consistent Structure

# Procedure Title

## Purpose
Why this procedure exists

## Prerequisites
- What's needed before starting
- Required access/permissions
- Required tools

## Procedure Steps
1. Step 1
2. Step 2
3. Step 3

## Verification
How to verify it worked

## Troubleshooting
Common issues and solutions

## Related Procedures
Links to related SOPs

## Last Updated
Date and author

2. Include Code Examples

## Running the Script

```powershell
.\Configure-Production-Firewall.ps1 `
    -Action Apply `
    -WireGuardIPRange "10.10.0.0/24"

- Use tools like ShareX, Greenshot

- Reference in markdown: `![Description](images/screenshot.png)`
