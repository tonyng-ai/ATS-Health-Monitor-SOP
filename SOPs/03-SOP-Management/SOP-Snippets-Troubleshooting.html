SOP Snippets Troubleshooting Guide

Issue: Snippets Not Working

If typing sop-procedure doesn't show the snippet, try these solutions:

Solution 1: Check VS Code Settings

Enable Snippet Suggestions

  1. Open VS Code Settings: Ctrl+,
  2. Search for: editor.snippetSuggestions
  3. Set to: "top" (snippets appear first)

Or add to .vscode/settings.json:

{
  "editor.snippetSuggestions": "top",
  "editor.tabCompletion": "on",
  "editor.suggest.showSnippets": true,
  "[markdown]": {
    "editor.snippetSuggestions": "top",
    "editor.tabCompletion": "on"
  }
}

Solution 2: Manually Trigger Suggestions

  1. Type sop-procedure
  2. Press Ctrl+Space to show suggestions
  3. Select the snippet from the list

Solution 3: Verify Workspace is Open

Important: VS Code workspace snippets only work when: - You opened the folder (not just a file) - File > Open Folder > Select I:\HyperV_PowerScript_Optimazation

Check: - Look at VS Code title bar - should show folder name - If it shows just filename, you opened a file, not the folder

Solution 4: Reload VS Code

  1. Press Ctrl+Shift+P
  2. Type: "Reload Window"
  3. Press Enter
  4. Try snippet again

Solution 5: Check Snippet File Location

Verify the file exists:

Test-Path .vscode\snippets\sop-snippets.code-snippets

Should return: True

Solution 6: Check File Language Mode

  1. Open your .md file
  2. Look at bottom-right of VS Code
  3. Should show: "Markdown"
  4. If not, click it and select "Markdown"

Solution 7: Alternative - Use User Snippets

If workspace snippets still don't work, copy to user snippets:

  1. Press Ctrl+Shift+P
  2. Type: "Preferences: Configure User Snippets"
  3. Select: "markdown.json"
  4. Copy contents from .vscode/snippets/sop-snippets.code-snippets
  5. Paste into user snippets file

User snippets location: - Windows: %APPDATA%\Code\User\snippets\markdown.json

Solution 8: Check for Conflicting Extensions

Some extensions (like Vim) can interfere with snippets: 1. Disable extensions temporarily 2. Test snippets 3. Re-enable one by one to find conflict

Verify Snippets Are Loaded

  1. Press Ctrl+Shift+P
  2. Type: "Preferences: Open User Settings (JSON)"
  3. Check if snippets are mentioned in output

Or check VS Code Output: 1. View > Output 2. Select "Log (Window)" 3. Look for snippet loading messages

Still Not Working?

Try this PowerShell command to verify everything:

# Check files exist
Write-Host "Snippets file exists: $(Test-Path .vscode\snippets\sop-snippets.code-snippets)"
Write-Host "Settings file exists: $(Test-Path .vscode\settings.json)"

# Check file content
Get-Content .vscode\snippets\sop-snippets.code-snippets | Select-Object -First 5

If all checks pass but snippets still don't work, the issue might be: - VS Code version (update to latest) - Extension conflict - Need to use user snippets instead
