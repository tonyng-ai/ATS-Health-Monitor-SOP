Purpose

This procedure covers the deployment, configuration, and maintenance of ATS 900 trading system with optimized JDBC connection pool settings to prevent database connectivity issues.

When to Use:

  • Deploying new ATS 900 instances
  • Troubleshooting "connection closed" errors
  • Configuring database connection pools
  • Setting up automated startup with visible terminals
  • Monitoring system health and performance

Prerequisites

Required Access

  • Administrator privileges (for scheduled task creation)
  • Read/Write access to C:\ATS\ats900\run\
  • Network access to database server: gfprod
  • SQL Server database access credentials

Required Tools

  • Java 8 (1.8.0_411 or compatible)
  • PowerShell 5.0 or later
  • SQL Server 2019 connectivity
  • Windows Server 2012 R2 or later

Required Information

  • Database server name: gfprod
  • Database name: fund2
  • Database credentials: ver2user / atsats
  • ATS ID: 900
  • JMX Port: 30313

Pre-Conditions

  • Java 8 is installed and in PATH
  • All required JAR files are present in lib\ directory
  • Network connectivity to database server is verified
  • Minimum 512MB JVM heap space available
  • Port 30313 is not in use by another process

Required Files Structure 

C:\ATS\ats900\run\
├── ats.ini (Configuration file)
├── ATS_Launcher.ps1 (Primary PowerShell launcher)
├── ATS_Launcher_Wrapper.bat (Wrapper for visible terminal in scheduled tasks)
├── Create-ScheduledTask.ps1 (Scheduled task creator)
├── Check-Java.ps1 (Java verification)
├── Check-JARs.ps1 (Dependency check)
├── Debug-ATS.ps1 (Troubleshooting)
├── GravPrimary_ATS.bat (Legacy batch file)
├── logs\ (Application logs)
├── atslog\ (ATS internal logs)
├── data\ (Application data)
└── lib\ (Java dependencies)

Step 1: Verify Prerequisites

Action:

# Navigate to ATS directory
cd C:\ATS\ats900\run

# Check Java installation
.\Check-Java.ps1

# Check JAR dependencies
.\Check-JARs.ps1

Expected Result:

  • Java 8 is installed and accessible
  • All required JAR files are present
  • No missing dependencies reported

Step 2: Configure ats.ini

Action:

Create or modify C:\ATS\ats900\run\ats.ini with the following settings:

# CONNECTION POOL CORE SETTINGS
db.pms.pool.size=30
db.rt.pool.size=40

# CONNECTION VALIDATION (CRITICAL)
db.pms.validationQuery=SELECT 1
db.rt.validationQuery=SELECT 1
db.pms.testOnBorrow=true
db.rt.testOnBorrow=true
db.pms.testWhileIdle=true
db.rt.testWhileIdle=true

# CONNECTION LIFECYCLE (PREVENT TIMEOUTS)
db.pms.maxLifetime=1680000    # 28 minutes (under 30min SQL Server default)
db.rt.maxLifetime=1680000
db.pms.idleTimeout=240000     # 4 minutes idle max
db.rt.idleTimeout=240000

# POOL MAINTENANCE
db.pms.timeBetweenEvictionRuns=30000    # Check every 30 seconds
db.rt.timeBetweenEvictionRuns=30000
db.pms.minEvictableIdleTime=180000      # 3 minutes minimum idle
db.rt.minEvictableIdleTime=180000

# FAIL-FAST TIMEOUTS
db.pms.connectionTimeout=10000          # 10 seconds max wait for connection
db.rt.connectionTimeout=10000
db.pms.loginTimeout=10
db.rt.loginTimeout=10

# DATABASE SERVER
db.pms.server=gfprod
db.rt.server=gfprod
db.pms.dbname=fund2
db.rt.dbname=fund2

Expected Result:

  • ats.ini file created/updated in C:\ATS\ats900\run\
  • All connection pool settings are configured correctly
  • Database server and database name are set

Step 3: Choose Deployment Method

Method 1: Manual PowerShell Launch (Testing)

Action:

# Navigate to ATS directory
cd C:\ATS\ats900\run

# Execute the launcher
.\ATS_Launcher.ps1

Expected Result:

  • PowerShell window displays launch information
  • Port 30313 is checked and available
  • ATS application starts in visible terminal
  • Java process begins initialization

When to Use:

  • Initial testing and verification
  • Troubleshooting sessions
  • Manual intervention required

Method 2: Scheduled Task Deployment (Production)

Action:

# Open PowerShell as Administrator
# Navigate to ATS directory
cd C:\ATS\ats900\run

# Execute the task creator with your username for visible terminal
.\Create-ScheduledTask.ps1 -UserName "$env:USERNAME"

⚠️ IMPORTANT - Visible Terminal Window:

  • SYSTEM account cannot show windows: Tasks running as SYSTEM account cannot display interactive windows to logged-in users (Windows security limitation)
  • Use user account for visible terminal: Always specify a user account with -UserName parameter to see the terminal window
  • Example: .\Create-ScheduledTask.ps1 -UserName "YourUsername"

Expected Result:

  • Scheduled task "ATS-900-Production" is created
  • Task is configured to run at system startup
  • Task is set to run with visible terminal window (when using user account)
  • Wrapper batch file (ATS_Launcher_Wrapper.bat) is created automatically
  • Task appears in Task Scheduler

When to Use:

  • Production deployments
  • Automated startup after reboot
  • Long-term operation
  • When terminal visibility is required for monitoring

Follow-up Steps:

  1. Verify task creation in Task Scheduler
  2. Test task execution manually (right-click → Run)
  3. Verify terminal window appears and shows ATS output
  4. Reboot system to test automatic startup

Note: If you need the task to run as SYSTEM (no visible window), omit the -UserName parameter. However, you will not be able to see the terminal output.

Method 3: Legacy Batch File (Fallback)

Action:

# Open Command Prompt
cd C:\ATS\ats900\run

# Execute the batch file
GravPrimary_ATS.bat

Expected Result:

  • ATS starts using legacy batch file method
  • Application initializes normally

When to Use:

  • PowerShell scripts unavailable
  • Emergency fallback method
  • Legacy system compatibility

Step 4: Monitor Startup

Action:

Monitor the terminal output for the following indicators:

  1. Port Check:

    • "Port 30313 is available" (green)

  2. Configuration:

    • "Launching ATS with connection pool configuration..."

  3. Application Startup:

    • Java process starts with assigned PID
    • No "connection closed" errors
    • Database connections initialize successfully
    • Symbols load with position initialization [0<>0]

Expected Result:

  • All startup checks pass
  • No error messages in output
  • Application reaches operational state

Verification

How to Verify Success

  1. Process Verification:

    Get-Process java | Where-Object { $_.Path -like "*java*" }

  2. Port Verification:

    Get-NetTCPConnection -LocalPort 30313 -ErrorAction SilentlyContinue

  3. Log Verification:

    Get-Content C:\ATS\ats900\run\logs\*.log -Tail 50

  4. Trade Commander Status:

    • Verify "OnLn" status is displayed
    • Confirm orders process normally

Success Criteria

  • Java process is running with PID
  • Port 30313 is in use by Java process
  • No "connection closed" errors in logs
  • Database connections initialize successfully
  • Symbols load with position initialization
  • Trade Commander shows "OnLn" status
  • No application crashes or freezes
  • Connection pool settings are active

Troubleshooting

Common Issues

Issue 1: Port 30313 Already in Use

Symptoms:

  • Error: "Port 30313 is in use by PID: [number]"
  • Cannot start ATS application

Solution:

# Find the process using the port
$process = Get-NetTCPConnection -LocalPort 30313 | Select-Object -ExpandProperty OwningProcess
Get-Process -Id $process

# Stop the process if it's a stale ATS instance
Stop-Process -Id $process -Force

# Verify port is now available
Get-NetTCPConnection -LocalPort 30313 -ErrorAction SilentlyContinue

Prevention:

  • Always stop ATS gracefully before restarting
  • Use Debug-ATS.ps1 to check for existing processes

Issue 2: Java Not Found

Symptoms:

  • Error: "Could not find or load main class java"
  • Java command not recognized

Solution:

# Run diagnostic script
.\Check-Java.ps1

# Verify Java installation
java -version

# Check PATH environment variable
$env:PATH -split ';' | Select-String -Pattern "java"

# If Java is installed but not in PATH, add it:
# [System.Environment]::SetEnvironmentVariable("PATH", $env:PATH + ";C:\Program Files\Java\jre1.8.0_411\bin", "Machine")

Prevention:

  • Install Java 8 and verify PATH during installation
  • Use Check-Java.ps1 before deployment

Issue 3: Connection Closed Errors

Symptoms:

  • Error: "The connection is closed"
  • Database connectivity issues
  • Intermittent connection failures

Solution:

  1. Verify ats.ini connection pool settings:

    Get-Content C:\ATS\ats900\run\ats.ini | Select-String -Pattern "pool|timeout|lifetime"

  2. Check database server connectivity:

    Test-NetConnection -ComputerName gfprod -Port 1433

  3. Review connection pool settings:

    • Ensure maxLifetime is under 30 minutes (1680000 ms = 28 min)
    • Verify testOnBorrow and testWhileIdle are true
    • Check idleTimeout is set to 240000 (4 minutes)

  4. Restart ATS with updated configuration

Prevention:

  • Use optimized connection pool settings from Step 2
  • Monitor connection pool metrics regularly
  • Set appropriate timeout values

Issue 4: Missing Dependencies

Symptoms:

  • Error: "NoClassDefFoundError"
  • Missing JAR file errors
  • Class not found exceptions

Solution:

# Run dependency check
.\Check-JARs.ps1

# Verify required JARs are present
$requiredJars = @(
    "..\atsv5.1.31tony.jar",
    ".\..\lib\mssql-jdbc-6.4.0.jre8.jar",
    ".\..\lib\log4j-core-2.24.3.jar",
    ".\..\lib\log4j-api-2.24.3.jar"
)

foreach ($jar in $requiredJars) {
    $fullPath = Join-Path "C:\ATS\ats900\run" $jar
    if (Test-Path $fullPath) {
        Write-Host "FOUND: $jar" -ForegroundColor Green
    } else {
        Write-Host "MISSING: $jar" -ForegroundColor Red
    }
}

Prevention:

  • Maintain complete lib\ directory
  • Use Check-JARs.ps1 before deployment
  • Document all required dependencies

Issue 5: Scheduled Task Terminal Window Not Visible

Symptoms:

  • Terminal window not appearing when task runs
  • Task executes but window is hidden
  • Cannot see ATS output or progress

Root Cause:

  • SYSTEM account limitation: Tasks running as SYSTEM account cannot display interactive windows to logged-in users. This is a Windows security feature (Session 0 isolation).

Solution:

Option 1: Recreate Task as User Account (Recommended)

# Delete existing task
Unregister-ScheduledTask -TaskName "ATS-900-Production" -Confirm:$false

# Recreate with your username
cd C:\ATS\ats900\run
.\Create-ScheduledTask.ps1 -UserName "$env:USERNAME"

Option 2: Verify Task Configuration

  1. Open Task Scheduler (taskschd.msc)
  2. Find "ATS-900-Production" task
  3. Right-click → Properties
  4. General tab:
    • Verify "Run only when user is logged on" is selected (for visible window)
    • OR "Run whether user is logged on or not" (for SYSTEM account - no window)
    • Check "Run with highest privileges"
  5. Actions tab:
    • Verify action uses cmd.exe with wrapper batch file
    • Should be: cmd.exe /c "C:\ATS\ats900\run\ATS_Launcher_Wrapper.bat"
  6. Settings tab:
    • Uncheck "Hidden" if present
    • Check "Allow task to be run on demand"

Option 3: Manual Launch for Monitoring
If you need to run as SYSTEM but want to see output:

# Keep scheduled task running in background (SYSTEM)
# Manually launch visible terminal when needed:
cd C:\ATS\ats900\run
.\ATS_Launcher.ps1

Prevention:

  • Always use -UserName parameter when creating scheduled task for visible terminal
  • Test task execution immediately after creation
  • Use Create-ScheduledTask.ps1 which automatically creates wrapper batch file
  • Remember: SYSTEM account = no visible window, User account = visible window

Emergency Recovery

Stop all ATS processes:

Get-Process java | Where-Object { $_.Path -like "*java*" } | Stop-Process -Force

Verify port availability:

Get-NetTCPConnection -LocalPort 30313 -ErrorAction SilentlyContinue

Restart using manual method:

cd C:\ATS\ats900\run
.\ATS_Launcher.ps1

Use Debug Script:

.\Debug-ATS.ps1

Maintenance Procedures

Daily Checks

  • Verify ATS process is running
  • Check Trade Commander status ("OnLn")
  • Review error logs for connection issues
  • Monitor database connection pool usage
  • Verify no "connection closed" errors

Script:

.\Debug-ATS.ps1 -QuickCheck

Weekly Maintenance

  • Rotate log files in logs\ directory
  • Verify disk space availability
  • Review ats.ini configuration
  • Test restart procedure
  • Check for Java updates (verify compatibility)

Script:

# Log rotation
$logs = Get-ChildItem C:\ATS\ats900\run\logs\*.log
$logs | Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-7) } |
    Compress-Archive -DestinationPath "C:\ATS\ats900\run\logs\archive_$(Get-Date -Format 'yyyyMMdd').zip"

Monthly Review

  • Analyze connection pool performance metrics
  • Review JDBC driver for updates
  • Verify database server connectivity
  • Update documentation as needed
  • Review and optimize connection pool settings

Notes

⚠️ WARNING:

  • Always verify port 30313 is available before starting ATS
  • Do not modify connection pool settings without understanding the impact
  • Test configuration changes in a non-production environment first

💡 TIP:

  • Use Debug-ATS.ps1 for comprehensive system diagnostics
  • Monitor connection pool metrics to optimize settings
  • Keep Java 8 installation separate from other Java versions to avoid conflicts

📝 NOTE:

  • Connection pool settings are optimized for SQL Server 2019 default timeout of 30 minutes
  • JMX port 30313 is used for monitoring and should not be exposed to public networks
  • Scheduled Task Visibility: To see terminal window, task MUST run as a user account (not SYSTEM). Use -UserName parameter when creating task.
  • Wrapper Batch File: ATS_Launcher_Wrapper.bat is automatically created to ensure visible terminal windows from scheduled tasks
  • SYSTEM Account Limitation: Tasks running as SYSTEM cannot show interactive windows to logged-in users (Windows Session 0 isolation)
