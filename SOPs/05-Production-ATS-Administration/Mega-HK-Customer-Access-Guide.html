Mega Hong Kong - ATS Health Monitor Access Guide

Version: 1.0.0 Last Updated: December 10, 2025 Classification: Customer Documentation Audience: Mega Hong Kong Team (External Customer)

Overview

This guide explains how Mega Hong Kong team members can securely access the ATS Health Monitor dashboard hosted in our NY datacenter.

What You'll Access: - ATS Health Monitor Dashboard: Real-time trading health monitoring - Position Tracking: Current and historical positions - Order History: FIX message audit trail - Email Notifications: Trade alerts and system notifications

Access Options Comparison

Option Security Complexity Best For
WireGuard VPN Highest Low Daily operations, multiple users
Cloudflare Tunnel High Very Low Browser-only access, no software install
Dedicated HK Instance Highest High Full autonomy, offline capability

Recommended: WireGuard VPN

For Mega HK, we recommend WireGuard VPN because: - End-to-end encryption - Fast connection (modern protocol) - Works from any network (office, home, mobile hotspot) - Simple one-time setup - No additional infrastructure in HK

Part 1: WireGuard VPN Setup (Recommended)

Step 1: Install WireGuard (One-Time)

Windows 10/11: 1. Download from: https://www.wireguard.com/install/ 2. Run the .msi installer 3. Click Next through all prompts 4. Restart if prompted

macOS: 1. Open App Store 2. Search for "WireGuard" 3. Install the app

Mobile (iOS/Android): 1. Search "WireGuard" in App Store / Play Store 2. Install the app

Step 2: Import Your Configuration

You will receive a configuration file from VCL IT team: - Filename: hk-mega.conf or similar - KEEP THIS FILE SECURE - it contains your private key

To import:

  1. Open WireGuard application
  2. Click "Import tunnel(s) from file"
  3. Select the .conf file provided
  4. Click Open

Step 3: Connect

  1. Select "hk-mega" (or your tunnel name)
  2. Click "Activate"
  3. Status should show: Active with green indicator

You're connected!

Step 4: Access the Dashboard

Open your browser and go to:

Service URL
ATS Health Monitor http://192.168.168.100:8080
SMTP4Dev (Email Viewer) http://192.168.168.100:8025

Login with your RBAC credentials provided by VCL IT.

Part 3: Best Practices

Security

  1. Never share your WireGuard config file
  2. Each user gets a unique configuration

  3. If compromised, we can revoke just that key

  4. Disconnect when not in use

  5. Click "Deactivate" in WireGuard when done

  6. Especially important on public WiFi

  7. Report lost/stolen devices immediately

  8. Contact VCL IT to revoke the key
  9. New key will be issued

Performance

  1. For best performance, use wired connection

  2. WiFi adds latency to the VPN

  3. If slow, try reconnecting

  4. WireGuard reconnects quickly

  5. Check "Latest Handshake" in WireGuard

  6. Should be < 2 minutes ago when active
  7. If older, reconnect

Part 4: Troubleshooting

Cannot Connect to VPN

Symptom Cause Solution
"Handshake: None" Network blocking UDP Check firewall allows UDP 51820
Handshake fails Server unreachable Check internet, try different network
Connected but no access Routing issue Disconnect and reconnect

Cannot Access Dashboard

Symptom Cause Solution
Page doesn't load VPN not connected Check WireGuard shows "Active"
"Connection refused" Service down Contact VCL IT
"401 Unauthorized" Bad credentials Check username/password
"403 Forbidden" No permission Contact VCL IT for role upgrade

Slow Performance

Symptom Cause Solution
Dashboard loads slowly High latency Use wired connection
Data not updating Connection dropped Reconnect WireGuard
Timeout errors Network congestion Try during off-peak hours

Appendix A: WireGuard Configuration Reference

Your config file looks like this (example):

[Interface]
PrivateKey = <your-unique-private-key>
Address = 10.101.0.40/32
DNS = 1.1.1.1

[Peer]
PublicKey = <VCL-server-public-key>
AllowedIPs = 10.101.0.0/24, 192.168.168.0/24
Endpoint = 209.127.118.182:51820
PersistentKeepalive = 25

Key fields: - Address: Your unique VPN IP (10.101.0.40-49 for HK team) - AllowedIPs: Networks you can access (already configured correctly) - Endpoint: VCL VPN server (NY datacenter)

Appendix C: Alternative - Cloudflare Tunnel (Browser Only)

If WireGuard cannot be installed (company policy restrictions), we can set up Cloudflare Tunnel as an alternative:

  • Works in any browser
  • HTTPS with valid certificate

Cons:

  • Slightly higher latency
  • Third-party involved (Cloudflare)
  • Additional cost

To request Cloudflare Tunnel access, contact VCL IT.
