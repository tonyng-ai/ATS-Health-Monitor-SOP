WireGuard Client Setup Guide

Purpose

This guide helps HK team members (and other authorized users) set up WireGuard VPN on Windows 10/11 to securely access ATS services.

Prerequisites

  • Windows 10 or Windows 11
  • Administrator access (for installation)
  • WireGuard configuration file (provided by IT)

Quick Start (5 Minutes)

Step 1: Download WireGuard

  1. Go to: https://www.wireguard.com/install/
  2. Click "Download Windows Installer"
  3. Run the downloaded .msi file
  4. Click Next through the installer

Step 2: Import Configuration

  1. Open WireGuard from Start Menu
  2. Click "Import tunnel(s) from file" (or Ctrl+I)
  3. Select the .conf file provided by IT (e.g., hk-mega.conf)
  4. Click Open

Step 3: Connect

  1. Select your tunnel from the list
  2. Click "Activate"
  3. Status should show "Active" with green indicator

You're connected! You can now access: - Plane (Issue Tracking): http://10.13.13.1:8000 - Other internal services as needed

Configuration File

You will receive a configuration file like this:

[Interface]
PrivateKey = <your-private-key>
Address = 10.13.13.2/32
DNS = 1.1.1.1

[Peer]
PublicKey = <server-public-key>
AllowedIPs = 10.13.13.0/24, 192.168.168.0/24
Endpoint = 209.127.118.182:51820
PersistentKeepalive = 25

DO NOT share this file - it contains your private key.

Status Check

When connected, you should see:

Field Expected Value
Status Active
Latest Handshake Less than 2 minutes ago
Transfer Shows rx/tx bytes

Troubleshooting

"Handshake did not complete"

Cause: Cannot reach VPN server

Solutions: 1. Check your internet connection 2. Verify firewall allows UDP port 51820 outbound 3. Contact IT - server may be down

"Invalid configuration"

Cause: Corrupted config file

Solutions: 1. Request new config file from IT 2. Ensure file was not modified

Connected but can't access services

Cause: Routing issue

Solutions: 1. Check "AllowedIPs" in config 2. Try disconnecting and reconnecting 3. Restart WireGuard application

Very slow connection

Cause: Network congestion or MTU issues

Solutions: 1. Add to [Interface] section: MTU = 1280 2. Try different network (home vs office)

Auto-Start (Optional)

To connect automatically when Windows starts:

  1. Open WireGuard
  2. Right-click your tunnel
  3. Check "Activate on boot"

Security Notes

  1. Never share your config file - each user has unique keys
  2. Report lost devices immediately - IT will revoke the key
  3. Don't modify the config - contact IT for changes
  4. Disconnect when not needed - especially on public WiFi
